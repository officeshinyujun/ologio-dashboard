# Ologio (Daily-Sunrin) 시연 시나리오 및 기능 연결 계획 (movie.md)

본 문서는 대시보드(웹)와 모바일 앱에 **실제 구현된 기능**을 기반으로 시연 영상을 촬영하기 위한 연출 시나리오입니다.
최근 작업을 통해 **백엔드에 구현되어 있었으나 앱에 연결되어 있지 않던 실시간 교시 맥락 API(`/timetable/me/context`)를 연동**하고, 일부 버그(시간표 템플릿 삭제 시 ID 대신 Name을 전송하던 이슈)를 수정하여 완성도를 높였습니다.

---

## 📊 기능 구현 및 연결 현황 요약

시연 진행 시 참고할 실제 데이터 연동 및 더미/모킹 상태입니다.

### 1. 백엔드 연동 완료 (실제 동작)
- **온보딩 및 프로필**: 구글 로그인(웹/앱), 학번 자동 파싱(`10101` 입력 시 1학년 1반 1번 설정), 프로필 수정 및 저장
- **실시간 교시 맥락**: 현재 교시 정보, 남은 시간, 과목, 교사, 강의실 및 이동 경로 표시 (`/timetable/me/context` 호출 후 네트워크 실패 시 클라이언트 자동 연산 Fallback 작동)
- **시간표 및 급식**: 주간 시간표 조회 및 보강/대체 수업 하이라이트, 일별/주간/월간 급식 조회, 대시보드에서 컴시간/급식 수동 동기화 요청
- **캘린더 및 수행평가**: 학사일정 및 과제 조회, 대시보드에서의 일정 생성(`School`, `Class` 등의 Scope 설정) 및 모바일 앱 실시간 반영
- **어드민 관리**: 사용자 목록 조회 및 등급 변경(Student ➡️ Admin), 과목 이름 매핑(NEIS 명칭 ➡️ 친근한 한글명 매핑), 일과 교시 템플릿 생성 및 수정/삭제

### 2. 시연용 모킹/시뮬레이션 (Mock)
- **푸시 알림 실시간 수신**: FCM 자격 증명이 누락된 환경을 고려하여, 대시보드에서 알림 발송 시 앱에서 로컬 알림(Local Notification) 형태로 수신되도록 시뮬레이션하거나 사전에 설정된 가짜 푸시 토큰으로 동작 유도
- **교내 WiFi 기반 텔레메트리**: 교외 촬영 환경을 고려하여 모바일 기기의 WiFi SSID 값을 `SUNRIN-WIFI`로 모킹하여 교내 접속 상태를 재현
- **구글 캘린더 실시간 정합성 복구**: 구글 API의 응답 속도 지연을 고려하여, 앱 내 수동 동기화 버튼 클릭 시 즉시 토스트 피드백을 노출하고 사전에 채워놓은 구글 캘린더 화면을 별도로 캡처하여 클로즈업

### 3. 미구현 및 추가 작업 태스크 (Tasks)
다음 기능들은 백엔드에 API가 없거나 프론트엔드에 설정 페이지는 존재하나 완전히 연결되지 않은 상태입니다. 시연 후 아래 내용으로 작업을 진행합니다.
1. **[앱] 구글 캘린더 연동 토글 스위치 활성화**: 프로필 설정의 구글 캘린더 연동 스위치가 현재 클라이언트더미 상태입니다. Revoke API 혹은 재인증 플로우 연결이 필요합니다.
2. **[앱/백엔드] 알레르기 목록 서버 동기화**: 현재 앱에서 선택한 알레르기 목록이 로컬 스토리지에만 저장됩니다. 유저 프로필 스키마에 알레르기 코드를 연동하는 API 및 프론트 작업이 필요합니다.
3. **[앱/백엔드] 알림 시간 설정 서버 동기화**: 로컬 스토리지에 저장되어 작동하는 급식/시간표 알림 시간 설정을 서버에 동기화하여 백엔드 푸시 스케줄과 맞추는 작업이 필요합니다.

## 📽️ 시연 시나리오 타임라인

### [0:00 ~ 1:00] 1단계: 대시보드 관리자 시점
1. **어드민 대시보드 진입**: 대시보드 홈 화면에서 사용자 수, GCal 동기화 율 등의 실시간 통계 차트를 보여줍니다.
2. **시간표/학사일정 동기화**: 설정 탭에서 NEIS/컴시간 수동 동기화를 작동시키고 동기화 로그 화면을 노출합니다.
3. **과목 매핑 수정**: 과목 매핑 탭에서 `정보보안실무_A`를 `정보보안 실무`로 가독성 있게 편집하여 저장합니다.

### [1:00 ~ 1:45] 2단계: 모바일 앱 첫 진입 및 온보딩
1. **구글 로그인**: 앱 첫 화면에서 구글 로그인을 수행하고 온보딩 화면으로 이동합니다.
2. **학번 입력**: 학번에 `10101`을 입력하고, 학년/반/번호가 자동으로 채워지는 모습을 시연합니다. 학과를 `소프트웨어과`로 선택하고 완료합니다.
3. **메인 홈 탭 진입**: 캘린더 준비 완료 토스트가 나타나며 깔끔한 대시보드 화면이 노출됩니다.

### [1:45 ~ 2:45] 3단계: 오늘의 스마트 루트 (모바일 앱)
1. **홈 화면 맥락 인지**: 실시간으로 연동된 `/timetable/me/context` API를 바탕으로 현재 교시 정보, 남은 시간, 강의실 및 다음 교시 이동 경로가 카드에 유려하게 노출되는 화면을 포커스합니다.
2. **시간표/급식 확인**: 시간표 탭으로 이동하여 보강 하이라이트를 확인하고, 급식 탭에서 당일 칼로리 및 알레르기 유발 물질 차단 설정을 시연합니다.

### [2:45 ~ 3:45] 4단계: 대시보드 ➡️ 앱 실시간 데이터 연동
1. **학급 수행평가 등록 (웹)**: 대시보드 일정 관리 탭에서 `인공지능 실무 수행평가`를 **1학년 1반 범위(Class Scope)**로 등록합니다.
2. **모바일 앱 실시간 반영**: 학생의 앱 캘린더 탭과 홈 탭 D-Day 영역에 `D-3 인공지능 실무 수행평가` 카드가 즉시 올라오는 모습을 화면 분할(Split)로 대조하여 연출합니다.
3. **어드민 전체 알림**: 대시보드에서 긴급 공지 알림을 발송하고, 모바일 폰 상단에 푸시 알림 배너가 떨어지는 연출을 진행합니다.

### [3:45 ~ 4:00] 5단계: 구글 캘린더 동기화 확인 및 마무리
1. **앱 프로필**: 설정 메뉴에서 'Google 캘린더 연동됨' 문구를 보여주고, 실제로 스마트폰 기본 구글 캘린더 앱에 접속하여 방금 등록한 `인공지능 실무 수행평가`가 반영된 것을 확인시켜 주며 마무리합니다.
